Database Coding Guidelines

Stored Procedures

  • Include author info, creation date, and description.
  • Use schema prefix (e.g., dbo).
  • Use SET NOCOUNT ON.
  • Use TRY...CATCH for error handling.

Tables

  • Use IF NOT EXISTS before creating tables.
  • Include schema prefix.
  • Use appropriate data types and identity columns.

Views

  • Use WITH VIEW_METADATA.
  • Avoid using CTEs.
  • Alias all columns.

Triggers

  • Include Skip Triggers logic.
  • Follow naming conventions (e.g., tICi, tICu).
